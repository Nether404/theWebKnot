## üìù Documentation Guidelines

### For Contributors

When adding new documentation:

1. **Choose the right location**:
   - User-facing guides ‚Üí `docs/guides/`
   - AI features ‚Üí `docs/` (root level)
   - Implementation details ‚Üí `docs/implementation/`
   - Test results ‚Üí `docs/testing/`
   - Obsolete docs ‚Üí `docs/archived/`

2. **Follow naming conventions**:
   - Use UPPERCASE for major docs (README.md, CHANGELOG.md)
   - Use descriptive names (REACT_BITS_INTEGRATION.md)
   - Use underscores for multi-word names

3. **Update this index**:
   - Add your document to the appropriate section
   - Update statistics if needed
   - Add to "I want to..." section if relevant

4. **Include metadata**:
   - Date created/updated
   - Author (if applicable)
   - Related documents

5. **Use consistent formatting**:
   - Markdown headers (##, ###)
   - Code blocks with language tags
   - Links to related docs
   - Clear section breaks
